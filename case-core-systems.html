基幹系システムの社内SE・業務改善(計7年)

期間: 計7年(2社) 領域: 販売・物流系基幹 技術: AS400 / RPG / EDI / Excel VBA チーム規模: 4〜15名 担当工程: 全フェーズ(要件定義〜保守運用)

案件概要

販売・物流系の基幹システム(AS400 / RPG)を対象に、社内SEとして 日々の運用対応、アドオン開発、業務改善、経営層への改善提案まで 幅広く担当した経験です。1つの現場だけでなく、複数の現場で同系統の 役割を担ってきました。

課題

  • 基幹系システムの変更依頼が日々発生し、業務部門の要望を仕様に落とし込む必要があった
  • 物流現場(受発注・検品・値札発行など)でのトラブル対応や運用改善が必要だった
  • 外注ベンダーへの依頼・導入調整を社内側で取りまとめる必要があった
  • 業務が属人化しやすく、改善提案にもとづいて仕組みへ落とし込むことが求められた

進め方

  • 業務部門からの要望を聞き、ユーザ業務を把握したうえでシステム仕様へ反映
  • アドオン開発(RPG)と、現場補助のための Excel VBA ツールを使い分け
  • ベンダーへの依頼内容、見積、導入テスト、切替までのコントロールを担当
  • 経営改革プロジェクトに参画し、役員層への改善提案と実行まで対応

担当範囲

2社にわたり計7年、社内SEとして継続的に従事。担当範囲は以下の通りで、いずれも期間中を通して対応しました。

  • 販売・受発注・物流領域における基幹システムの運用・改善・アドオン開発
  • EDI 対応、ハンディ端末、物流センター業務支援、外販向け物流サービスの運用改善
  • 業務改善に伴う要件定義、基本設計、手順書・提案書の作成
  • ベンダーコントロール(依頼・導入・検収までの一連の調整)
  • 経営層への改善提案、改革プロジェクトへの参画

意識したこと

  • 「システム側の都合」ではなく、現場業務を起点に仕様を組み立てる
  • 運用の継続性を前提に、止めずに少しずつ変えていく
  • 外注先・ユーザ・経営層の間で、用語と判断基準をそろえる
  • 改善提案で終わらせず、導入・定着までを担当する

今から見て

この時期に扱ったのは古い基盤ですが、得られたのは技術そのものよりも、 「業務理解を起点にシステム側を動かす進め方」でした。現在の Web 系案件でも、 この社内SE時代の視点が、要件整理や既存業務との接続に活きています。 同じように、現場とシステムの間を整理してつなぎ直す関わり方ができます。
