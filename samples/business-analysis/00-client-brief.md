# 業務整理パック サンプル｜クライアント前提

> このドキュメントは、業務整理パックの納品物の質感を見ていただくためのサンプルです。架空の中小企業を想定して作成しています。

## 想定クライアント概要

| 項目 | 内容 |
| --- | --- |
| 企業名（仮） | 株式会社マルキ食品 |
| 業種 | 食品卸（酒類・調味料・乾物が中心） |
| 所在地 | 関東圏の地方都市 |
| 設立 | 1978年 |
| 従業員数 | 28名（事務6・営業10・配送8・倉庫4） |
| 年商 | 約12億円 |
| 取引先 | 飲食店・小売店 約200社 |

## 現状の業務とシステム

- 注文受付：FAX（約6割）／電話（約3割）／LINE・メール（約1割）
- 受注処理：事務担当がExcelに手で転記、ピッキング指示書を印刷して倉庫へ
- 在庫管理：Excel（リアルタイム性なし、夜間に手動更新）
- 顧客マスタ：旧販売管理システム（オンプレ）と kintone に二重管理
- 請求業務：専用パッケージソフトで発行、紙で郵送
- 配送ルート：ベテラン配送社員2名の経験で組まれており、属人化している

## 経営者の問題意識（ヒアリング想定）

- 「ベテラン配送が定年を迎えるが、ルート知識を引き継げていない」
- 「FAX転記のミスでクレームが月3〜5件出ている」
- 「kintone を入れたが、旧システムとの二重入力が止まらない」
- 「AIで何かできるはずだが、何から手をつければよいか分からない」
- 「外注したが要件が固まらず、見積も出てこなかった」

## このパックで対応する範囲

- 現状業務の棚卸しと業務フロー図化
- 課題の分類と原因の整理
- 改善案の提示（AI活用の是非を含む）
- 着手順の優先順位付け

## 対応しない範囲

- 実装・改修そのもの（パック後の別契約として対応可能）
- 業務システムの選定・購買代行
- 社内の人事・組織設計

## 期間・進め方（標準）

- 期間：2〜3週間
- 工程：ヒアリング90分×2回 → 整理 → 中間共有 → 納品
- 中間共有時に方向性のすり合わせを行い、納品時の手戻りを防ぐ
