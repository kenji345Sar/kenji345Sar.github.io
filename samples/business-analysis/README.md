# 業務整理パック サンプル成果物

業務整理パックの納品物の質感を見ていただくためのサンプル一式です。架空の中小企業（食品卸・従業員28名）を想定して作成しています。

## このサンプルの目的

「業務整理パックを依頼すると、どんな成果物が手元に残るのか」を、実例ベースで把握していただくためのものです。実案件では、これをクライアント固有の事情に合わせて作成します。

## 含まれるドキュメント

| # | ファイル | 内容 |
| --- | --- | --- |
| 00 | [client-brief.md](00-client-brief.md) | 想定クライアント前提（業種・規模・現状システム・経営者の問題意識） |
| 01 | [business-flow.md](01-business-flow.md) | 業務フロー図（1枚／PNG画像） |
| 02 | [issues.md](02-issues.md) | 課題整理メモ（5件 + 3つの構造的原因への俯瞰） |
| 03 | [proposals.md](03-proposals.md) | 改善案3本（AI活用の是非を明示） |
| 04 | [priority.md](04-priority.md) | 優先順位付き対応案（3フェーズ／意思決定ポイント／費用感） |

## 読む順序

上から順（00 → 04）に読むと、業務整理の思考プロセスをそのまま追えます。

- **時間がない方は 04 だけ**でも、改善案の全体像と着手順は把握できます
- **業界・課題感が近い方**は 00 → 02 で痛点の整理を、03 で AI 活用の判断軸を確認してください

## このサンプルの特徴

- **AI 活用の是非を明示している**：3本の改善案のうち、AI が強く効くもの・部分的に効くもの・効かないものを分けて記述しました
- **着手順を意思決定ポイントで区切っている**：3フェーズに分けることで、途中での修正・中断が可能な構成
- **第1期で1ヶ月以内に成果を出す設計**：早期成果で次フェーズへの判断材料にする方針

## 実案件との違い

このサンプルは架空のクライアント設定です。実案件では以下が異なります。

- 業種・規模・既存システムが固有に
- ヒアリングでしか出てこない経営者個別の論点が反映される
- 改善案の「効果見積」「費用見積」が具体数値で出る
- 顧客の予算・体制制約に応じた現実的な優先順位になる
