# 配送日報Web化 サンプル成果物

Excel・紙運用のWeb化の納品物の質感を見ていただくためのサンプル一式です。
[業務整理パックサンプル](../business-analysis/README.md)と同じ架空クライアント（食品卸「株式会社マルキ食品」）を想定し、**整理の結果、先行で着手することにした配送日報のWeb化**を題材にしています。

## このサンプルの目的

「Excel・紙運用のWeb化を依頼すると、どんな成果物が手元に残るのか」を、要件定義から段階リリース計画まで一連で示すためのものです。

業務整理パックのサンプルが**ドキュメント納品物**だとすれば、こちらは**実装に入る直前の設計成果物**です。実案件では、これをもとに実装フェーズへ進みます。

## 含まれるドキュメント

| # | ファイル | 内容 |
| --- | --- | --- |
| 00 | [requirements.md](00-requirements.md) | 要件定義メモ（現状・ゴール・対応範囲・想定スタック） |
| 01 | [screen-design.md](01-screen-design.md) | 画面設計（入力／一覧／CSV出力 のワイヤー＋設計思想） |
| 02 | [data-design.md](02-data-design.md) | データ設計（テーブル定義・権限・移行方針） |
| 03 | [ai-usage.md](03-ai-usage.md) | AI活用ポイント（要約／自然文検索／異常検知の効き方） |
| 04 | [release-plan.md](04-release-plan.md) | 段階リリース計画（6週間・3版・意思決定ポイント） |

## 読む順序

00 → 04 の順で、Web化の思考プロセスを追えます。

- **時間がない方は 04 だけ**でも、リリース計画とAIの入れ方の方針が把握できます
- **実装イメージを知りたい方**は 01 → 02 で画面とデータの粒度を確認してください
- **AIをどこまで使うかが気になる方**は 03 で「効く／部分的／効かない」の分類を見てください

## このサンプルの特徴

- **AIを第1版に入れない設計**：転記廃止と蓄積基盤を先に確立し、データが揃ってからAIを差し込む
- **既存Excelと1件もズレない移行**：CSV互換と並行運用を最重要要件に据える
- **業務整理パックとの連動**：本サンプルの第3版が、業務整理パック第1期（配送ルート知識のAI抽出）の素材になる構造

## 業務整理パックサンプルとの関係

両サンプルは同じ架空クライアントを共有しており、**整理 → 実装** の流れを通しで読むことができます。

- [業務整理パックサンプル](../business-analysis/README.md)：論点整理から優先順位付き対応案まで（ドキュメント納品物）
- 配送日報Web化サンプル（本ディレクトリ）：整理の結果、先行で着手した実装の設計成果物

実案件では、業務整理パックを起点に、こうした実装フェーズへ自然に接続します。

## 実案件との違い

このサンプルは架空のクライアント設定です。実案件では以下が異なります。

- 業種・規模・既存システム（kintone／FileMaker／オンプレ販売管理など）が固有に
- 既存Excel書式の細部やマクロの動きが反映される
- 工数見積・費用見積が具体数値で出る
- 顧客の体制・予算制約に応じた現実的なリリース計画になる
