# 配送日報Web化 サンプル｜画面設計

紙とExcelで回っていた配送日報を、スマホ／タブレット／PCのいずれでも使えるWebアプリに置き換えます。
画面は3つに絞り、**配送員はほぼ「入力画面」だけを使う**シンプル構成にしています。

## 画面一覧

| # | 画面 | 主な利用者 | デバイス |
| --- | --- | --- | --- |
| S1 | 日報入力 | 配送員 | スマホ／タブレット |
| S2 | 日報一覧・検索 | 事務／管理者 | PC（タブレットも可） |
| S3 | CSV出力（既存Excel書式互換） | 事務 | PC |

## S1. 日報入力画面（配送員向け）

[01-input-screen.svg](assets/01-input-screen.svg)

### 設計の考え方

- **入力項目を最小化**：配送員は片手・グローブ越し・休憩中に打つ前提
- 配送先・商品マスタは**プルダウンと予測候補**で選ぶだけ。手入力は「特記事項」だけに集約
- 数量・時刻はステッパー／ピッカーで入力ミスを物理的に減らす
- 一度提出したあとも当日中は再編集可能（紙時代の「あとで赤ペン直し」相当）

### 入力フィールド

| 項目 | 入力方式 | 必須 |
| --- | --- | --- |
| 日付 | 自動（当日） | ◎ |
| 配送員 | 自動（ログイン） | ◎ |
| 便（午前／午後／臨時） | ボタン選択 | ◎ |
| 配送先（複数） | マスタ選択＋予測候補 | ◎ |
| 各配送先の品目・数量 | マスタ選択＋ステッパー | ◎ |
| 出発／帰庫時刻 | タイムピッカー | ◎ |
| 走行距離 | 数値入力（前回値からの差分予測） | ○ |
| 積み残し | チェック＋自由記入 | ○ |
| 顧客対応メモ | 自由記入（音声入力対応） | ○ |
| 異常・ヒヤリハット | チェック＋自由記入 | ○ |

### モバイル特化のポイント

- 1画面1セクション、縦スクロールのみ。横スクロール禁止
- ボタンは親指で押せる最小サイズ（44×44px以上）
- 通信が切れても入力は継続できるように、ローカル保存→復帰時に同期

## S2. 日報一覧・検索画面（事務／管理者向け）

[01-list-screen.svg](assets/01-list-screen.svg)

### 設計の考え方

- **「あの日の、あの便、あの顧客の話」**にすぐ辿り着けることが最優先
- 一覧は表形式。フィルタは画面上部に固定し、検索結果は同じ画面で絞り込み
- AI全文検索（後述）は検索バーから自然文でも投げられる

### 主な機能

- 日付・配送員・便・配送先での絞り込み
- 自然文検索：「先週の積み残しが出た便」「○○商店の苦情があった日」
- 異常・ヒヤリハットがある日報は**赤バッジ**で目立たせる
- 行クリックで詳細表示（読み取り専用、管理者のみ編集可）

## S3. CSV出力画面

[01-csv-export.svg](assets/01-csv-export.svg)

### 設計の考え方

- 移行コストを最小化するため、**既存Excelの月次集計シートと同じ列順**でCSVを吐く
- 期間（開始日〜終了日）と配送員でフィルタしてダウンロード
- 「文字コード（Shift_JIS／UTF-8）」を選べる（既存Excelマクロが Shift_JIS 前提のため）

### 出力例

```csv
日付,配送員,便,配送先コード,配送先名,品目,数量,異常,メモ
2026-04-01,佐藤,午前,K0102,○○食堂,日本酒一升瓶,12,,通常
2026-04-01,佐藤,午前,K0118,△△レストラン,醤油5L,6,積み残し,在庫切れ
...
```

> **既存Excelとの互換**は、業務移行を止めないための最重要要件。「Web化したらExcel連動が壊れた」を絶対に起こさない設計です。

## 画面遷移

```
[ログイン]
   ↓
   ├─ 配送員 → [S1: 日報入力] (＋自分の過去日報閲覧のみ)
   │
   └─ 事務／管理者 → [S2: 日報一覧] ⇄ [日報詳細]
                        ↓
                     [S3: CSV出力]
```

## アクセシビリティ・UI方針

- 文字サイズ：本文16px以上。配送員画面はさらに大きめ
- コントラスト比：WCAG AA 準拠（屋外で見ても読める）
- 色のみで情報を伝えない（異常マークはアイコン＋色＋テキスト）
