# 配送日報Web化 サンプル｜段階リリース計画

「全部入りで半年」ではなく、**2週間ごとに使える物が増える**設計です。
途中で止めてもそれまでの分は業務改善として残ります。

## 全体像（約6週間）

```
週: 1   2   3   4   5   6
   ┌───────┐
   │ 第1版 │  入力＋一覧＋CSV出力／配送員2名で先行運用
   └───────┘
           ┌───────┐
           │ 第2版 │  全員展開＋AI要約・自然文検索
           └───────┘
                   ┌───────┐
                   │ 第3版 │  異常検知ハイライト・運用磨き込み
                   └───────┘
```

## 第1版（〜2週目末）：転記廃止が成立する最小構成

### スコープ

- S1 日報入力（スマホ／タブレット）
- S2 日報一覧・検索（フィルタ絞り込みのみ、自然文検索なし）
- S3 CSV出力（既存Excel書式互換）
- ロール（driver / office / admin）と権限
- 配送員2名で先行運用、紙日報と並行

### この時点での価値

- **事務の転記作業が「先行運用ぶん」だけ消える**
- 配送員側は紙とスマホ両方に書く必要があるが、**1日数分の追加で済む設計**にしておく
- 並行運用で**Webと紙の差分を毎日チェック**し、漏れた項目があれば即修正

### 意思決定ポイント①（第1版終了時）

以下のいずれかが満たせなければ、第2版に進まず**第1版を磨き込む**判断:

- 配送員2名がストレスなく入力できているか（ヒアリング）
- CSV出力が既存Excel月次集計と1件もズレなく一致するか
- 当日中の入力完了率が90%以上か

## 第2版（3〜4週目）：全員展開とAI第1段

### スコープ

- 配送員全員（8名）への展開
- 紙日報の廃止（並行運用終了）
- A1 日報要約（日次・週次・ドライバ別）
- A2 自然文での過去日報検索

### この時点での価値

- 事務の転記作業が**ゼロ**になる
- 管理者は朝、**前日サマリだけ読めば全便の状況把握ができる**
- 「先週の積み残し」「○○商店の苦情があった日」を**30秒で引き出せる**

### 意思決定ポイント②（第2版終了時）

- 全員が当日中にWeb日報を出せているか
- AI要約の内容を管理者が「使える」と評価しているか（5段階で平均3.5以上目標）
- 自然文検索のヒット率が体感で7割を越えているか

## 第3版（5〜6週目）：異常検知と運用磨き込み

### スコープ

- A3 異常検知ハイライト（数値閾値＋AI傾向検知）
- 月次レポートPDF生成（経営会議向け）
- 配送員からの改善要望対応（UIチューニング）
- ベテラン配送員の**ドライバ別サマリ**を引き継ぎ資料化

### この時点での価値

- ベテラン引退時の引き継ぎ資料が**自動的に蓄積されている状態**になる
- 異常の早期発見が仕組みとして回り始める
- 業務整理パック第1期「配送ルート知識のAI抽出」と**データが連動する**

### 終了時の引き渡し物

- 稼働中のWeb日報システム
- 操作マニュアル（配送員用／事務用／管理者用）
- データバックアップ手順とサーバ運用ガイド
- 第3期への接続ポイント整理（業務整理パック第3期データ連携の事前準備）

## リスクと対策

| リスク | 兆候 | 対策 |
| --- | --- | --- |
| 配送員がWeb入力を嫌がる | 当日中入力完了率が下がる | 第1版から先行2名で慣らし、UIフィードバックを取る／紙と並行で2週間 |
| CSVが既存Excelと一致しない | 月次集計でズレが出る | 第1版完了時に**過去1ヶ月分を再生成して旧Excelと完全一致確認** |
| AI要約が外し続ける | 管理者が読まなくなる | プロンプトと出力レビューを毎週／使われていない場合は**潔く止める判断** |
| AI APIのコストが想定外 | 月次費用が予算超過 | 第2版投入時に1週間分の実コストを計測してから全展開 |

## 業務整理パックとの接続

| 業務整理パック | 配送日報Web化 |
| --- | --- |
| 第1期：配送ルート知識のAI抽出 | **第3版の「ドライバ別サマリ」がそのまま素材になる** |
| 第2期：受注統合＋AI標準化 | 別系統（並行進行可） |
| 第3期：受注→出荷→請求のデータ連携 | **第3版完了後、日報データが連携対象になる** |

> 配送日報Web化は、業務整理パックで描いた**3期計画の補強と先行**として位置づけています。
> 「整理した結果、これも先にやっておくと効く」が見えたから先行する、という典型的な流れです。

## このサンプルから読み取っていただきたいこと

- **全部入りで半年**ではなく、**2週間で使える物が出る**段階リリースで作る
- 各フェーズの終わりに**意思決定ポイント**を置き、止めるか進むかを判断できる
- AIは**蓄積データができてから後乗せ**。先にAIを入れるとデータ不足で精度が出ない
- 既存のExcel運用と**1日も1件もズレない移行**を最重要要件に据える
