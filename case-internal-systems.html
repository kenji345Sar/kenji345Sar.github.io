社内業務の効率化・連携対応

期間: 約9ヶ月 技術: Laravel 8 / PHP / kintone / VBA / JavaScript / Oracle / MySQL チーム: 3名 担当工程: 要件定義〜保守運用(全工程)

案件概要

業務ごとに Excel・紙・独立したシステムが混在し、二重入力や手作業が 発生していた状態から、申請・稟議系業務の Web 化と、既存業務・既存システム との連携整理を進めた案件です。

課題

  • 申請・稟議系の業務が紙や Excel 中心で、回覧や履歴追跡に手間がかかっていた
  • 経費精算や押印など、業務ごとに独立した運用になっており、データが分散していた
  • 既存のレガシーシステム(PHP4 / Oracle)の保守も並行して必要で、完全入れ替えは現実的ではなかった

進め方

  • 現行業務を整理し、どこをローコード(kintone)で、どこを Laravel API で扱うかを仕分け
  • 既存業務を止めないことを前提に、小さく切り出して順に Web 化
  • API・バッチ処理・VBA など、複数の手段を用途ごとに使い分けて連携

担当範囲

計 4つの業務テーマ を並行で担当。ローコード(kintone)と API 開発(Laravel)、レガシー保守を用途に応じて使い分けました。

  • 事業部門管理資料のバッチ処理化(Laravel API 作成、VBA 側の対応)
  • 経費精算まわりの見直し(既存ツールと kintone との連携)
  • 押印・稟議運用の Web 化(紙運用から kintone への移行)
  • 既存レガシーシステム(PHP4 / Oracle)の保守対応

意識したこと

  • ローコードと API 開発の「使い分け」。kintone で十分なものは無理に作らない
  • 既存業務の運用リズムを崩さないよう、段階的に移行する
  • レガシー保守と新規開発の並行を前提に、共有できる部分は揃える

今から見て

こういう案件の価値は、Web 化そのものではなく、業務と既存システム・ 既存運用をつなぎ直す部分にあると考えています。同じように 「全部入れ替えは難しいが、このまま回すのも厳しい」という状況に対して、 小さく整えていく関わり方ができます。
